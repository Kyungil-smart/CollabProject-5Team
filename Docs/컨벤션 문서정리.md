# 그라운드 룰 { 초안 작성중 }
- 핵심 규칙은 모든 팀원이 합의하고 숙지하여 철저하게 지켜야할 사항입니다
- 권장 사항은 개인의 판단으로 유연하게 대처할 수 있는 영역입니다
---

## 핵심 규칙
1. 풀 리퀘스트 작성 규칙
- 브랜치 병합시 develop에 풀 리퀘스트 작성
- 풀 리퀘스트 요청시 작업 내역을 요약하여 팀원에게 알림
---

2. 작업테스크 관리 및 문서 작성
- 작업예정 업무, 작업중인 업무, 그리고 작업 완료한 업무까지 모두 공유
- 작업 내역에 대한 문서 및 회고를 작성하여 팀원과 공유
---

3. 프로젝트 구조 파악
- 프로젝트 핵심 구조를 파악하고 사용
- 폴더 구조를 숙지하고 활용
---

4. 작업영역에 대한 책임감 가지고 테스트 하기
- 병합전에 콘솔 로그를 확인하고 크리티컬한 버그가 없는지 테스트
- 다양한 예외 상황을 직접 검증하며 주요 병합작업 후에도 추가 검증 진행
---

5. 충돌 방지
- 타인의 코드나 공동 사용중인 에셋 변경시 팀원에게 알리기
- 같은 씬을 두명의 작업자가 동시에 작업하지 않으며, 복사하여 작업 또는 프리펩 활용
- 프리펩 수정시 override 해야하는 상황(원본수정)과 아닌 상황(복사본/바리언트) 구분
- Tag, Layer 변경시 팀원에게 알림
---
---
## 권장 사항
1. 브랜치 네이밍
- 네이밍 작성 방식 통일: 접두사/기능_작성자 (예: feat/player_PEY)
- 사용하지 않는 오래된 브랜치는 지워 혼동을 방지
- 풀 리퀘스트에서 코드 리뷰 요청 가능
---

2. 커밋 메시지 규칙
- 접두사 feat, fix, refactor 등 작성 [참고링크](https://udacity.github.io/git-styleguide/)
---

3. 코드 네이밍 및 주석 규칙
- 변수/매개변수: camelCase, 클래스/메서드/구조체/열거형: PascalCase, 상수: UPPER_CASE
- private의 경우 앞에 _ 언더바를 붙여서 표기한다
- 인터페이스: I로 시작, 추상클래스: Base로 끝남, 스크립터블 오브젝트: SO로 끝남
- 네이밍 규칙을 준수하고 코드의 작명을 명확하게 하여 가독성을 높이기
- 클래스당 최소 하나의 summary 또는 주석을 사용하여 요약하고 설명하기
---

4. 프리펩 네이밍 규칙
- 대분류는 폴더트리 이름으로 잡고 세부는 '속성_이름_특이사항' 에서 최소 2가지 이상으로 정리
- 예: [Prefabs/Characters/Enemy] 폴더의 Orc_Boss, [Prefabs/UI] 폴더의 Player_Hpbar_Blue)
---

5. 데이터 정리
- 객체마다 데이터에 차이를 주는 경우, ScriptableObject로 데이터 정리
- Resources.Load 보다 어드레서블 사용 권장
- Imports 폴더의 에셋 공유는 Google Drive 사용
 ---

6. 네임 스페이스 사용
- 핵심 코드를 제외한 나머지는 기능단위의 간단한 네임스페이스를 사용
--- 

## 예외 사항
- 긴급한 수정이나 사소한 변경의 경우 main에 커밋할 수 있지만, 팀원에게 알리기
---

## 초기 폴더 구조
```
Docs/              문서정리 폴더
Assets/
├─ Imports/         외부 에셋 (gitignore 대상)
├─ Packages/        외부 패키지 (R3등.. 경로 의존)
├─ Project/          핵심 폴더
│  ├─ Art/
│  │  ├─ Audio/
│  │  ├─ Materials/
│  │  ├─ Shader/
│  │  ├─ Sprites/
│  │  └─ Texture/...
│  ├─ DB/               스크립터블 오브젝트
│  ├─ Prefabs/
│  ├─ Scenes/
│  └─ Scripts/
│     └─ Core/...
├─ Settings/         유니티 세팅
│  ├─ Editor/           에디터 전용 스크립트 (빌드제외)
│  ├─ Input/            통일된 InputSystem 세팅
│  ├─ TextMesh Pro/
│  └─ URP/...
└─ WIP/              테스트/정리전 잡다한거 방치해도 무방한 폴더
```
- 외부 에셋은 Imports에 두지만, 설정 변경등 공유가 필요하면 드라이브에 공유 또는 Project/Art등에 정리(문제 없을 경우)
- 폴더는 필요에 따라 추가
