# 플레이어 이동 및 상호작용

## 설정
- 바닥
  - 레이어 설정: Ground Layer
  - 컴포넌트: Collider, NavMesh Surface 부착 후 'Bake'
  
- 책상(데스크탑 프리팹 만들어 둠): 
  - 레이어 설정: Default Layer
  - 컴포넌트: Collider(Is Trigger체크), DeskInteract.cs ('업무 시작' 버튼을 누르면 띄워줄 UI창, 충돌할 Player Layer설정)

- NPC
  - 레이어 설정: Interactable Layer
  - 컴포넌트: Collider, NPCInteract.cs(NPC 이름 설정, NPC대화 UI창, UI내부 TMP, Button 드래그 앤 드랍)
  
- 플레이어
  - 레이어 설정: Player Layer
  - 컴포넌트: PlayerMove.cs, NavMesh Agent, Rigidbody(Is Kinematic 체크) PlayerMove컴포넌트에'레이어 설정', '상호작용 거리' 설정

## 구현한 로직

### IInteractable.cs에 인터페이스로 작성하여 상호작용 가능한 오브젝트별로 구현할 수 있도록 함
  1. `OnInteract()`함수로 상호작용을 한 뒤 무엇을 할건지
   
  2. `GetTransform()`함수로 상호작용할 오브젝트의 위치를 Player에게 전달해서 이동할 수 있게 함
  
### PlayerMove.cs에서는 직접적으로 플레이어의 움직임과 상호작용을 담당
  1. 유니티 에디터에서는 클릭, 모바일 빌드환경에서는 터치를 입력받아 NavMeshAgent와 연동하여 캐릭터를 이동시키며 Legacy Input System을 사용함
   
  2. 대화창 등의 UI가 이미 열려있는 상태 (hasInteracted == true)라면 터치 입력을 차단하여 움직임을 제어
   
  3. 터치한 지점에 UI버튼 위이면 `IsPointerOverUI()`를 통해 움직임 Raycast가 발사되지 않도록 함
   
  4. Ground 레이어를 터치하면 기존 타겟 정보를 초기화하고 해당 좌표로 이동
   
  5. Interactable 레이어를 터치하면 타겟의 Collider와 IInteractable인터페이스를 변수에 저장, 사물의 정중앙 좌표를 향해 이동
   
  6. 상호작용 타겟쪽으로 이동 시 중심점이 아닌 콜라이더 표면의 가장 가까운 지점(ClosestPoint)과 플레이어 사이의 실제 직선 거리를 매 프레임 계산
   
  7. 설정한 사거리이내로 들어가면 `ResetPath()`로 멈추고 캐릭터가 기울어지지 않게 Y축을 고정한 채 LookAt으로 해당 오브젝트 방향으로 회전

### DeskInteract.cs(트리거 기반 상호작용)
- Player가 해당 영역에 들어가면 작동

- OnTriggerEnter: Player Layer가 붙은 오브젝트가 트리거 영역에 들어오면 하단 UI에 'Work Start' 버튼 활성화(`SetWorkStartButtonInteractable(true)`)

- OnTriggerExit: 플레이어가 영역을 벗어나면 'Work Start' 버튼 비활성화

### NPC와의 대화 상호작용
- IInteractable 인터페이스를 상속받은 NPC오브젝트와의 상호작용 시, 당일 업무 및 퀘스트 진행 상태에 따라 대화 콘텐츠를 분기하고 퀘스트 완료 처리.
  - 일일 업무 완료 전까진 일반대화만 가능
  - 퀘스트 완료(Quest Complete)버튼을 클릭하기 전까지 특별대화 유지
  - 퀘스트 완료버튼을 누른 뒤로는 일반대화가 나옴

- NPC와 대화 상호작용
  -  하이어라키창의 이름이 npcName으로 Start()시점에 자동할당
  -  npcName이 DateTimeManager.cs에 HashSet으로 저장되어 각각의 NPC이름별로 상호작용 가능하도록 구현
  -  NPC프리팹을 복사해서 하이어라키 창에서 이름만 바꿔주면 됨

- 플레이어가 NPC에게 `OnInteract()` 시도하면 `DateTimeManager.Instance.GetDialogueState(npcName)`을 호출하여 현재 조건에 맞는 상태 코드(int)를 반환
  - 상태 코드 0 : 당일 업무 미완료 (일반 대화 : 'HI'로 해둠)
  - 상태 코드 1 : 당일 업무 완료 후 첫 대화(퀘스트 완료버튼 누르기 전까지 동일) (특별 대화 : 'Quest'라고 해둠)
  - 상태 코드 2 : 당일 NPC퀘스트 완료 버튼 누른 뒤 (일반 대화: 'See Ya'라고 해둠)

- 이벤트 리스너 가비지 누적을 막기 위해, `Start()`에서 `OnClickQuestComplete()`를 한 번만 등록

- '퀘스트 완료' 버튼을 클릭하는 시점에 DateTimeManager 내의 (`talkedNpcsToday`)에 NPC이름이 기록 되어 상태 2로 전환
