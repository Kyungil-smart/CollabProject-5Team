# 낮 밤 시스템

## 개요
- 주차, 요일, 낮/밤 시스템을 구현하여 일일업무완료 상태와 연동하여 퇴근 루프 구현
- R3를 사용하여 데이터의 변경사항이 UI컴포넌트에 실시간 자동으로 반영되도록 구조 채택

## 구현한 로직

### EData.cs
- Enum타입 순수 C# 스크립트에 요일 (`DayOfWeek`), 낮/밤(`TimeOfDay`)
  - DayOfWeek : Monday ~ Friday
  - TimeOfDay : Day, Night

### DateTimeManager.cs
- 날짜 상태를 R3의 기능인 `ReactiveProperty`로 들고 있으며, 이 값이 바뀌면UI에 갱신됨
  - ReactiveProperty<int> currentWeek (기본값 : 1주차)
  - ReactiveProperty<DayOfWeek> currentDay (기본값: 월요일)
  - ReactiveProperty<TimeOfDay> currentTime (기본값: 낮)

- 일일 상태 제어 및 리셋(`ResetDayStatus()`)
  - 업무 완료 상태를 결정하는 변수 `isWorkCompleted = false` 선언
  - 당일 특별 대화를 마친 NPC이름을 저장하는 HashSet<string> `talkedNpcsToday` 선언
  
- 퇴근, 날짜 흐름(`OnClickEndDayButton()`)
  - 업무가 완료되지 않았다면 퇴근 처리 못함(`!isWorkCompleted` Return)
  - 퇴근 버튼 클릭 시 현재 요일/시간에 따라 분기 처리 후 마지막에 `ResetDayStatus()`를 호출해 초기화함
    1. 금요일 낮 퇴근 -> 금요일 밤으로
    2. 금요일 밤 퇴근 -> 다음 주 월요일 낮으로
    3. 월 ~ 목 낮 퇴근 -> 다음 날 낮으로

### 낮/밤 변경 및 퇴근 로직을 테스트해보기 위한 DateTimeTestUI.cs 사용법
- 해당 스크립트를 오브젝트에 부착 후 컴포넌트들을 드래그 앤 드롭으로 연결
  - TMP
    -  Date Text : 주차, 요일, 낮,밤 이 표기될 TMP
    -  Work Status Text 오늘 업무 완료 여부를 보여줄 TMP
  - 버튼
    - Work Start Button : 업무 시작 Button
    - Get Off Work Button : 퇴근 Button
  - UI, 참조
    - Task UI : '업무 시작 버튼' 을 누르면 켜지고 완료 시 닫혀야 하는 UI창
    - Work Complete Button : TaskUI 내에 있는 '업무 완료' 버튼

#### DateTimeTestUI.cs에 구현된 로직
- R3기반 데이터 자동 동기화 (`Start()`)
  - DateTimeManager의 currentWeek, currentDay, currentTime을 구독
  - 해당 스크립트에서 값이 바뀌면 인스펙터에 연결된 TMP에 반영됨

- 업무 상태별 버튼 활성화 제어(`UpdateUI()`)
  -  isWorkCompleted값에 따라 실시간으로 상태 텍스트와 퇴근 버튼의 활성화

- 책상 상호작용 (`SetWorkStartButtonInteractable()`)
  - 외부 센서(DeskInteract에서 호출, 업무 완료 전일 때 '업무 시작' 버튼활성화/비활성화)

- 퇴근 시 연동 예외처리
  - 퇴근 버튼을 누르면 다음 날짜로 넘어가기 전, 초기화
    - deskUI(업무 UI창) 비활성화
    - 플레이어가 UI창이 켜져있을 때 이동하지 못하는 플래그(`hasInteracted = true`)를 해제하기 위해 `PlayerMove.CloseInteractionUI()`를 호출해 이동 로직 원복
    - 그 후 `DateTimeManager.Instance.OnClickEndDayButton()`을 호출해 날짜 전환
