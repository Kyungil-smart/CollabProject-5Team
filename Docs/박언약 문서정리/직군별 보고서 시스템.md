# 작업날짜
- 5/28: '직군별 보고서 시스템' 문서 생성

## 개요
- 금요일 밤 ProgressNight 시점에 curProject에 투입된 직원 데이터로 보고서 등급/종류를 산정하고,
승인 대기 목록에 올린 뒤, 직군(Part)당 1개씩 채택 시 curProject의 progress를 올리는 시스템.
---

## 전체 흐름
```
[금요일 밤 ProgressNight 호출]
  → Project.GenerateReportDrafts()
      → 파트(Planning / Develop / Art)별 투입 직원을 순회
      → 직원 1명당 Report 1건 생성
          - score 계산 (직원 능력치 + motivation 보정)
          - score 구간으로 ReportGrade 결정 (INNOVATION / STANDARD / SLOPPY)
          - Report.GetRandomReport(part, grade)로 실제 ReportSO 매칭
      → 생성된 Report전체를 Project.pendingReports에 저장
  → UI가 pendingReports를 파트별로 표시 (플레이어 선택 화면)
  → 플레이어: 파트당 1개 선택 → Project.SelectReport(draft) 호출
  → 플레이어: '승인' 버튼 클릭 → Project.ApproveSelectedReports() 호출
      → selectedReports 내 각 Report의 등급에 따라
          해당 파트 점수(qualityScore / stabilityScore / charmScore) 증가
      → pendingReports / selectedReports 초기화
```
---

## 상세 단계별 설명

### 1단계 — 직원 점수 계산 (ReportPolicy.CalcScore)
```
직원 기본 점수 = (property1 + property2 + property3) / 3
motivation 보정:
  motivation >= 80  → +5
  motivation 40~79  → +0
  motivation < 40   → -10
최종 score = 기본 점수 + 보정값
```

### 2단계 — ReportGrade 결정 (ReportPolicy.CalcGrade)
```
score >= 70  → INNOVATION (최고)
score >= 45  → STANDARD   (보통)
score <  45  → SLOPPY     (부실)
```

### 3단계 — ReportSO 매칭
```
Report.Instance.GetRandomReport(employee.so.partParsed, grade)
  → (Part, ReportGrade) 키로 reportMap에서 무작위 1개 반환
  → null이면 해당 직원의 ReportDraft는 생성 스킵
```

### 4단계 — 승인 후 progress 기여 계산 (ReportPolicy.CalcContribution)
```
등급별 점수 기여:
  INNOVATION  → contribution = score × 1.5f
  STANDARD    → contribution = score × 1.0f
  SLOPPY      → contribution = score × 0.5f

파트 → 프로젝트 점수 매핑:
  Planning → qualityScore   +contribution
  Develop  → stabilityScore +contribution
  Art      → charmScore     +contribution
```

> qualityScore + stabilityScore + charmScore = progress (CombineLatest로 자동 반영)

## 클래스 책임 분리

| 클래스 | 역할 |
|---|---|
| `ReportPolicy` (static) | 점수 계산 / 등급 산정 / contribution 계산 공식 |
| `ProjectSO` (SO) | 보고서 초기값, 시트데이터 연동 |
| `Report` (class) | 직원 1명당 생성되는 런타임 보고서 인스턴스 (so, author, score) |
| `ReportManager` (singleton) | SO 레지스트리, `GetRandomReport(part, grade)` 제공 |
| `Project` | pendingReports / selectedReports 보관, Generate / Select / Approve 메서드 소유 |
| `Company` | ProgressNight 시점에 → 프로젝트와 연동 → 보고서 연동|

## 미구현 
- 특성(Trait) 보정: ReportSO.mainTrait 기반으로 contribution 추가 증감
- 복수 프로젝트 동시 진행 시 각 Project 독립적으로 처리
- 직원 피로도 연동: 승인된 직원이라면 피로드 증가 아니면 감소
